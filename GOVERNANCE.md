# Governance

Nestra is stewarded by DIP Dev with an open, consensus‑seeking development model.

## Roles
* Maintainers: Approve architecture, review PRs, manage releases & roadmap.
* Contributors: Anyone with merged commits (thank you!).
* Users: Provide feedback, file issues, join discussions.

## Decision Making
* Small changes: PR + at least one maintainer approval.
* Larger design shifts: `proposal` issue → discussion → consensus → PR.
* Ties / unresolved conflicts: final call by a maintainer.

## Adding Maintainers
Evaluated on sustained quality contributions, constructive reviews, and community conduct.

## Inactivity
Maintainers inactive for 6+ months may be moved to emeritus (reversible).

## Security & Sensitive Issues
Handled privately per `SECURITY.md` policy.

## Changes to Governance
Amended via PR with clear rationale and community review.
